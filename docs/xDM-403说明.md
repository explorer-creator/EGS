# xDM 相关接口出现 403 时

## 原因

`/api/dynamic/api/...` 会由代理转发到 **xDM-F（默认 8003）**。若 xDM 要求登录会话，而请求未带有效 **Cookie / Token**，会返回 **403 Forbidden**。

本机 **Spring Security** 已对 `/api/**` 放行，**403 一般不是代理本身拦截**，而是 xDM 返回的。

## 处理方式（二选一）

### 1. 使用 Mock（推荐本地演示，默认已开）

在 `proxy/src/main/resources/application.yml` 中：

```yaml
xdm:
  mock-mode: true   # 或通过环境变量 XDM_MOCK_MODE=true
  # remote-entities 留空：全部实体仍按 mock-mode 决定 Mock 或 8003
  remote-entities: ""
```

此时设备、物料、工序等接口**不访问 8003**，返回内置演示数据，**不应再出现 xDM 403**。

### 1b. 按实体拆分：部分 Mock、部分走 8003

当需要 **多数模块继续 Mock**，只有 **个别 xDM 实体**（如用户 `User_Infor`）连真实服务时，配置 **白名单**：

```yaml
xdm:
  mock-mode: true          # 全局仍可保持 true
  remote-entities: User_Infor,Feedback   # 或环境变量 XDM_REMOTE_ENTITIES=User_Infor,Feedback
```

- **`remote-entities` 非空**：仅列表中的**实体名**（与动态 API 路径一致，如 `EquipmentManagement`、`MaterialManagement`）会转发到 **8003**；**未列入的实体始终 Mock**，不受 `mock-mode` 影响。
- **`remote-entities` 为空**：行为与原来相同，**全部**由 `mock-mode` 决定。

### 2. 连接真实 xDM（全部实体）

1. 设置 `XDM_MOCK_MODE=false`（或 `mock-mode: false`），且 **`remote-entities` 留空**。
2. 启动 xDM-F（8003）并登录控制台。
3. 在页面顶部 **「xDM Cookie」** 文本框粘贴浏览器里该会话的 **Cookie**（见界面说明），点击保存。

## 修改配置后

请**重启** Spring Boot 代理（`mvn spring-boot:run` 或你的启动方式），配置才会生效。
