# EGS 智慧制造前端项目 — 阿里云部署指南（2GB 服务器场景）

> 适用对象：将本仓库 `EGSfb/front`（Vue 2 + Vite）及关联后端/可选服务部署到阿里云。  
> 文档版本：1.0 · 更新日期：2026-03  
> **配套文件：** 同目录下 `Aliyun-Deployment-Guide.docx` 为本文的 Word 版（内容一致，便于打印与修改格式）。

## 一、项目与运行时依赖（部署前必读）

| 组件 | 说明 | 生产环境常见做法 |
|------|------|------------------|
| **前端** | `EGSfb/front`，Vite 构建为静态文件 | Nginx / OSS 静态托管 |
| **Java 代理/API** | `vite.config.js` 将 `/api` 代理到 `http://localhost:8080` | 同一台 ECS 或内网另一台运行 JAR |
| **模块 A（可选）** | Python FastAPI `EGSfb/A/server.py`，端口 8000 | 按需部署；不配则前端「智能物料配图」用本地演示图 |
| **绿能排产演示（可选）** | Next.js `greengeneratesysteam` | 独立进程或另一台小机；前端通过 iframe 嵌入 |

生产环境一般不再使用 Vite 开发服务器，而是：**Nginx 托管 `dist` + 反向代理 `/api` 到 Java**。

## 二、2GB 内存能否够用？

**结论：可以跑「前端静态 + 一个 Java 进程」的轻量生产，但偏紧，需做取舍。**

| 场景 | 2GB 是否可行 | 说明 |
|------|--------------|------|
| Nginx + 静态资源 + 单个 Spring Boot（堆内存 512MB～768MB） | ✅ 常见 | 关闭不必要服务，监控内存 |
| 再叠加 Next.js + Python + MySQL 同机 | ⚠️ 吃力 | 建议拆机或使用托管数据库 |
| 高并发、大 JVM、ElasticSearch 同机 | ❌ 不建议 | 需升配或拆分 |

**建议 JVM 参数示例（按实际调整）：**

```text
-Xms256m -Xmx768m -XX:+UseG1GC
```

## 三、推荐阿里云产品组合（按预算从简到全）

### 方案 A：最低成本（个人/演示）

| 产品 | 规格建议 | 用途 |
|------|----------|------|
| **轻量应用服务器** 或 **ECS** | **2 vCPU / 2GB** 起，带宽 **3～5 Mbps** | 统一部署 Nginx、前端静态、Java 后端 |
| **弹性公网 IP**（若未自带） | 固定 IP 便于域名解析 | 绑定到实例 |
| **域名** + **备案**（大陆机房） | — | 网站访问与 HTTPS |

可选：**SSL 证书**（免费 DV 证书即可，阿里云可申请）。

### 方案 B：更稳妥（小团队/需数据库）

在方案 A 基础上增加：

| 产品 | 规格建议 | 用途 |
|------|----------|------|
| **云数据库 RDS**（MySQL/PostgreSQL） | 入门规格 | 替代同机自建库，省内存、易备份 |
| **对象存储 OSS** | 按量 | 静态资源、导出文件、备份包 |
| **CDN** | 按量 | 加速全国访问（静态站效果明显） |

### 方案 C：前后端分离、可扩展

| 产品 | 用途 |
|------|------|
| **SLB 负载均衡** | 多台 ECS 时入口统一、健康检查 |
| **容器服务 ACK**（可选） | 容器化部署，运维成本高，2GB 单机通常不必上 |

**针对本项目的务实建议：**

1. **首上云**：**ECS 或轻量 2G** + **域名备案** + **Nginx + HTTPS**。  
2. **Java 与数据库**：若后端强依赖数据库，**RDS 入门版**比同机装 MySQL 更省 ECS 内存。  
3. **前端**：构建后的 `dist` 可只放在 ECS；访问量上来再加 **OSS + CDN**。  
4. **绿能 Next / Python**：演示环境可**不部署**或单独一台**最低配**实例，避免拖垮 2G 主站。

## 四、部署流程概览

```
本地/CI 执行 npm run build
        ↓
上传 dist 到服务器（如 /var/www/egs）
        ↓
Nginx 指向静态目录，并 proxy /api → 127.0.0.1:8080
        ↓
Java 代理监听 8080（systemd 托管）
        ↓
安全组放行 80/443，配置 SSL
```

## 五、服务器环境准备（Alibaba Cloud Linux / CentOS / Ubuntu 示例）

### 5.1 安装 Nginx

```bash
# Alibaba Cloud Linux / CentOS
sudo yum install -y nginx

# Ubuntu
sudo apt update && sudo apt install -y nginx
```

### 5.2 安装 Java 运行环境（若部署 JAR）

```bash
# 示例：OpenJDK 11
sudo yum install -java-11-openjdk -y   # 或按发行版选择
```

### 5.3 构建前端（在开发机或 CI）

```bash
cd EGSfb/front
npm ci
npm run build
```

产物目录：`EGSfb/front/dist`。

### 5.4 生产环境变量

根据 `src/config.js`，构建时可使用 `.env.production` 或流水线环境变量，例如：

```env
VITE_API_BASE=/api
VITE_TENANT_ID=你的租户ID
VITE_APPLICATION_ID=你的应用ID
VITE_GREEN_DEMO_URL=https://你的绿能演示域名或留空
```

重新 `npm run build` 后上传新的 `dist`。

## 六、Nginx 配置示例

将 `dist` 解压到 `/var/www/egs`，示例站点：

```nginx
server {
    listen 80;
    server_name your.domain.com;
    root /var/www/egs;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        # 若后端需要鉴权头，按 proxy 项目要求保留，例如：
        # proxy_set_header X-Auth-Token $http_x_auth_token;
    }

    # 可选：模块 A Python，若同机部署在 8000
    # location /api/module-a/ {
    #     proxy_pass http://127.0.0.1:8000/;
    #     proxy_set_header Host $host;
    # }
}
```

启用 HTTPS 建议使用 **阿里云 SSL 证书** 或 **Let’s Encrypt**，在 Nginx 中增加 `listen 443 ssl` 与证书路径。

## 七、Java 后端（proxy）常驻

假设打包为 `proxy.jar`：

```ini
# /etc/systemd/system/egs-proxy.service
[Unit]
Description=EGS API Proxy
After=network.target

[Service]
Type=simple
User=ec2-user
WorkingDirectory=/opt/egs
ExecStart=/usr/bin/java -Xms256m -Xmx768m -jar /opt/egs/proxy.jar
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now egs-proxy
```

**说明：** 实际 JAR 名称、端口、环境变量以 `EGSfb/proxy` 项目为准；若端口不是 8080，需同步修改 Nginx `proxy_pass`。

## 八、阿里云控制台必做配置

### 8.1 安全组

| 方向 | 协议 | 端口 | 源 |
|------|------|------|-----|
| 入方向 | TCP | 22 | 仅你的办公 IP（SSH） |
| 入方向 | TCP | 80 | 0.0.0.0/0（HTTP） |
| 入方向 | TCP | 443 | 0.0.0.0/0（HTTPS） |

**不要**对公网开放 8080、3306、6379 等，除非有明确需求并限制来源 IP。

### 8.2 域名与备案

- 使用**中国大陆节点**的 ECS/OSS 网站服务，域名需 **ICP 备案**。  
- 备案期间可先通过 **IP** 或 **未备案域名指向境外节点** 做测试（以阿里云规则为准）。

### 8.3 镜像与地域

- 选择与用户最近的 **地域**（如华东）。  
- 镜像选 **Alibaba Cloud Linux 3** 或 **Ubuntu 22.04 LTS** 等长期支持版本。

## 十、2GB 机器上的「不要做」清单

1. 同一台 2G 上同时跑 **重型中间件**（如 Elasticsearch、大数据组件）。  
2. **不设 swap** 且同时开多个 JVM / Node 生产进程。  
3. 生产环境长期用 `npm run dev` 对外提供服务。  
4. 把 **数据库 root** 暴露到公网。
