# EGS 智慧制造与物流平台 — 录屏解说词（约 10 分钟）

> **使用说明**：按左侧菜单自上而下 → 右侧菜单自上而下 → 全局组件的顺序操作；语速适中、关键处可停顿。**字数约 2900 字**，口播约 9～11 分钟（视操作停顿而定）。  
> **技术主线**：SPA 前端、BFF 统一推理代理、**RAG 检索增强对话**、多模态与运筹可视化、数字孪生、可信展示。

## 【片头 · 0:00 — 0:45】整体架构与入口

各位评委、老师好。下面我演示的是 **EGS 智慧制造与物流一体化平台**。  
从工程上看，这是一个典型的 **单页应用 SPA**，基于 **Vue 2.7 与组件化** 构建；前后端采用 **前后端分离**，浏览器只通过 **同源 `/api` 路径** 访问后端，由 **Spring Boot 承担 BFF——也就是 Backend for Frontend** 的职责：统一做 **鉴权、跨域策略和大模型推理代理**，避免把 **API Key** 暴露在前端，这符合企业里 **最小权限** 和 **密钥收敛** 的实践。

请先登录或访客进入主界面。大家可以看到 **双栏导航**：左侧偏 **制造与贸易主题**，右侧是 **工厂、供应链与物流运营域**，中间是 **主内容区**，并配合 **keep-alive** 做部分页面的 **状态缓存**，减少反复加载时的抖动。下面我们按菜单顺序走一遍核心能力。

---

## 【左侧栏 · 0:45 — 3:30】制造、贸易与智能生成

**第一组：平台与知识展示。**  
打开 **「平台介绍」**，这里承担 **品牌叙事与业务边界** 说明。接着 **「行星减速器原理」** 与 **「工序与工艺介绍」** 是 **制造域的知识内容层**，为后面的 **结构化问答** 做语义铺垫。

**第二组：大语言模型落地——产品制造问答。**  
进入 **「产品制造问答」**。这一模块调用后端的 **大语言模型推理**，走的是 **OpenAI 兼容协议**，由 Java 代理转发到 **云侧推理端点**。我输入一个产品名称，系统会生成 **原材料、设备、工序、工艺路线** 等 **结构化知识输出**。这里体现的是 **认知智能** 在 **工艺规划辅助** 场景的应用；若后端未配置密钥，也会有 **明确降级提示**，这是 **可观测性** 的一部分。

**第三组：视觉与绿色制造。**  
**「视觉质检」** 展示 **计算机视觉在工业质检** 中的占位与扩展位，可对接 **目标检测、OCR** 等云端或边缘 API。  
**「绿色生产」** 与 **「绿能排产演示」**：后者采用 **iframe 嵌入队友的 Next 子系统**，体现 **异构技术栈集成** 与 **微前端式组合**，避免重复造大屏，符合 **开放集成** 而不是 **单体堆功能** 的思路。

**第四组：经营分析与贸易洞察。**  
**「商业分析」** 侧重 **描述性分析与策略文案** 的扩展位，子工程可按 **OpenAI 兼容后端** 接入 **经营推理**。  
**「贸易可视化」** 使用 **ECharts** 对 **进出口商品结构与区域分布** 做 **描述性分析可视化**，数据来自项目内 **摘要数据集**，强调 **统计口径与可信标注**，而不是实时海关明细。

**第五组：可信与跨境。**  
**「AI+链集成」** 把 **人工智能输出** 与 **链上存证叙事** 做 **故事层衔接**。  
**「丝路班列协同」** 体现 **跨境班列、仓线协同** 的大屏与流程。  
**「生成综合报告」** 展示 **结构化导出能力**，便于 **路演与归档**。

**第六组：生成式配图——智能物料配图。**  
打开 **「智能物料配图」**。这条链路是 **爬取候选图像 + 质量门控**；当爬取失败或不达标时，由 **Python FastAPI 侧车服务** 调用 **文生图 T2I** 做 **生成式兜底**，在架构上属于 **与主 JVM 进程隔离的 Sidecar**，承担 **I/O 密集与生成任务**，符合 **故障隔离** 里的 **舱壁模式** 思想。

## 【右侧栏 · 3:30 — 8:00】工厂、供应链与智慧物流核心

**智能工厂与智慧供应链。**  
展开 **「智能工厂」**：**设备孪生·台账**、**工序编排·执行**，对应 **OT 域设备与过程** 的数字化占位。  
展开 **「智慧供应链」**：**物料中枢·主数据**、**工艺路线·编排**，体现 **主数据与工艺** 的衔接。

**赛题核心——智慧物流。**  
请重点停留 **「智慧物流」**。这一页是 **人工智能加智慧物流** 的 **集中展示**：

- **WMS 仓储与分拣波次**：演示 **库位、出入库流水、波次任务队列**，状态可 **本地持久化**，体现 **仓储执行系统** 的交互闭环。  
- **「物流 AI 接口映射」** 子页：我们把业界常见的 **视觉、NLP、运筹优化、预测分析** 四类能力，映射到 **典型 API 形态**，体现 **体系化理解**，不是单点炫技。  
- **「路径规划 A*」**：在 **栅格地图** 上运行 **A* 最短路**，并支持 **多停靠点顺序** 的 **启发式近似**，对应 **AGV / 分拣机器人路径** 的教学级原型；真实生产可替换为 **地图 ETA API** 或 **OR-Tools 等求解器**。  
- **「AI+物流 3D 沙盘」**：基于 **WebGL 与 Three.js** 的 **数字孪生视窗**，包含 **AGV 路径动画、物联指标、吞吐可视化**；右侧还有 **SHA-256 链式摘要** 的 **动画演示**，以及浏览器端 **AES-128-CBC、RC4** 的 **货物信息加解密 PoC**，对应 **传输保密与内容摘要** 的 **可信叙事**。

若时间紧，可在本页 **略扫各 Tab**，但 **务必口头点出**：**运筹确定性算法 + 大模型非确定性推理** 在同一产品里的 **分工**。

**仓储与运输。**  
**「仓库调度」**、**「智能 WMS」** 延伸 **库内作业** 场景。  
**「智慧 TMS」** 展示 **运单、ETA、延误解释**：其中延误说明可经 **大模型生成自然语言根因分析**，未配置时 **规则 Fallback**，体现 **SLA 可演示** 与 **服务降级策略**。  
**「多模态报单」** 是 **多模态融合** 的典型入口：**视觉语言模型 VLM** 解析 **运单截图**，**文本模型** 解析 **自然语言报文**，未配置时 **规则槽位填充**，体现 **Slot Filling** 的工程路径。

**地图与特色场景。**  
**「运输地图」** 做 **地理可视化**；**「爱宠专送」** 体现 **垂直场景化产品化** 能力。  
**「使用指南」** 与 **「架构先进性」** 可在录屏尾声 **快速扫过**，作为 **可运维性与技术说明** 的文档入口。

## 【全局壳层 · 8:00 — 9:15】智能体与丝路贸易态势

**右侧快捷区** 提供 **站内搜索、模型导出、用户抽屉**，体现 **运营侧工具化**。

**小机器人智能体** 全站可拖拽，本质是 **对话入口与意图路由** 的 **统一壳**。这里值得单独讲一句 **检索增强生成 RAG**：用户发问时，后端会 **优先调用 `/api/rag/chat`**——先在 **本地领域知识库** 里做 **检索与片段拼装**，再把 **带上下文的提示词** 送给 **通义千问**。这样做的好处是：**回答更贴我们工业 / 物流领域的术语与定义**，**减轻纯大模型「一本正经胡说」的风险**，同时 **知识条目可维护、可替换为向量库 Top-K**，属于 **企业级问答** 常见演进路径。若 RAG 链路异常，系统会 **自动回退到原来的直连对话接口**，保证 **演示与答辩的鲁棒性**。

**重点：丝路贸易态势。**  
点击屏幕最右侧的 **「丝路贸易态势」** 竖条，打开 **全屏抽屉**。默认第一个 Tab 已 **嵌入完整「智慧物流」页面**，实现 **控制塔式体验**：**同一入口既看物流业务全景，又切换「国际线路 3D 地球」与「国内低空物流地图」**。3D 地球使用 **WebGL 与弧线路由示意**；低空视图使用 **Leaflet 与深色底图**，体现 **全球—国内多尺度可视化**。

## 【收尾 · 9:15 — 10:00】技术亮点归纳

最后做 **四点** **技术亮点归纳**，方便评委抓关键词：

1. **推理工程化**：**BFF 统一代理、OpenAI 兼容契约、密钥不出前端**，面向 **企业级 LLM 落地** 的最小可行安全模型。  
2. **运筹与孪生**：**确定性路径算法 + 三维数字孪生 + 指标与链式摘要可视化**，覆盖 **决策—执行—可视化** 闭环。  
3. **集成与扩展**：**Python Sidecar 配图、iframe 异构子系统、可选经营分析服务**，体现 **可演进架构** 而不是 **一次性演示**。  
4. **RAG 与可信对话**：**检索增强** 让 **领域知识先进模型、再生成**，突出 **可解释、可维护** 的 AI 应用方式；配合 **回退策略**，体现 **工程上的稳妥**，而不是只追「单次回答炫技」。
