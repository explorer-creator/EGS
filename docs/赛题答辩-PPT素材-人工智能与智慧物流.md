# EGS 智慧制造与物流平台 — 答辩级技术白皮书（PPT 素材）

> **定位**：面向竞赛答辩与路演的高层技术叙事，侧重 **架构语义、能力边界与差异化价值**，而非操作手册。  
> **技术栈锚点**：Vue 2.7 + Vite 前端；Spring Boot **统一推理与业务代理层**；FastAPI 异步配图服务；可选独立 **经营分析微服务**。  
> **赛题锚定**：**人工智能 × 智慧物流** —— 以 **大语言模型（LLM）+ 多模态理解（VLM）+ 运筹可视化 + 数字孪生** 贯通 **计划—执行—履约—可视化** 闭环。

## 二、架构视角：分层与责任边界（建议作为「系统架构」页）

### 2.1 逻辑分层（C4 语境下的 Container 级）

```
┌─────────────────────────────────────────────────────────────┐
│  Presentation Layer                                          │
│  SPA（Vue）· 组件化大屏 · WebGL（Three.js）· 地图（Leaflet）   │
└────────────────────────────┬────────────────────────────────┘
                             │ HTTPS · 同源 /api 反向代理
┌────────────────────────────▼────────────────────────────────┐
│  Edge / BFF Layer（Spring Boot）                             │
│  · 统一鉴权与 CORS 策略                                       │
│  · LLM 推理代理（兼容 OpenAI Chat / Vision JSON Schema）       │
│  · 业务 API 聚合与脱敏                                       │
└────────────────────────────┬────────────────────────────────┘
                             │
         ┌───────────────────┼───────────────────┐
         ▼                   ▼                   ▼
┌─────────────────┐ ┌───────────────┐ ┌─────────────────────┐
│ Cloud Inference│ │  Geo Services │ │  Python Sidecar     │
│  (DashScope)   │ │  (可选)       │ │  FastAPI · 异步 I/O  │
│  LLM / VLM /T2I│ │               │ │  爬取管线 + T2I 兜底  │
└─────────────────┘ └───────────────┘ └─────────────────────┘
```

**术语要点**

- **BFF（Backend for Frontend）**：前端不直连大模型公网端点，由 Spring 层 **收敛密钥**、**统一观测与限流**（工程上可演进为 **API Gateway + Key Vault** 模式）。
- **Sidecar 模式**：Python 配图服务 **进程隔离**，避免阻塞主链路 JVM，符合 **故障隔离（Bulkhead）** 思想。
- **OpenAI-Compatible Contract**：降低 **模型供应商切换成本**，上层仅依赖 **HTTP + JSON** 契约，而非 SDK 锁定。

### 2.2 数据与智能的「语义层」示意

- **结构化业务数据**：WMS 库位、波次、TMS 运单等 —— 前端状态 + 本地持久化（演示级）。
- **非结构化多模态输入**：运单截图、自然语言报文 —— 经 **VLM + LLM** 解析为 **槽位填充（Slot Filling）** 结果。
- **运筹结果可视化**：A\* 路径、多停靠点顺序 —— **确定性算法** 输出，可与 **地图 ETA API** 或 **求解器（OR-Tools 等）** 替换对接，体现 **算法可插拔**。
- **RAG 管线**：在 **BFF** 内完成 **「检索 → 拼接 System 上下文 → 调 LLM」**，与 **纯直连 LLM** 形成 **降级回退**（`RobotAgent` 先 RAG 后 `/api/llm/chat`），兼顾 **演示稳定性** 与 **能力进阶**。

## 五、创新点与壁垒叙事（评委 Q&A 备用）

1. **控制塔式体验**：将 **计划层（贸易结构）— 执行层（仓配）— 可视化层（3D/地图）** 压合在同一产品壳内，符合 **供应链控制塔** 的产品化表达。  
2. **推理层工程化**：**密钥不上屏、统一代理、兼容协议**，体现 **企业级 LLM 落地** 的 **最小可行安全模型（Minimum Viable Security）**。  
3. **运筹可替换**：栅格路径与启发式序列为 **可验证的确定性核心**；地图与 VRP 求解器可 **热插拔**，避免「黑箱演示」。  
4. **可信叙事闭环**：加密 + 摘要链 **可视化**，与 **AI+链** 模块形成 **故事一致性**。  
5. **异构集成**：**iframe 契约集成** 第三方大屏，体现 **开放平台** 思维而非单体堆叠。  
6. **RAG 落地**：**检索与生成解耦**（`init_knowledge.json` / 未来 **Qdrant**），**同一套 DashScope 密钥** 完成 **Embedding 与推理** 扩展，符合 **企业知识库** 演进路径。

## 六、技术架构一页话（可直接上墙 / 脚注）

> 前端 **SPA** 经 **同源 /api** 访问 **Spring Boot BFF**；大模型调用 **仅在后端出站**，遵循 **最小权限密钥管理**；配图 **异步 Sidecar** 承担 **I/O 密集与生成式兜底**；全栈 **演示级数据** 与 **可选真实推理** 分离，保障 **答辩可复现**。
