# 智慧物流系统中的人工智能 API 分类

业界实践常将智慧物流相关 AI 能力归为四类：**视觉识别**、**自然语言处理**、**运筹优化**、**预测分析**。以下为典型场景与接口形态说明（含第三方服务示例，选型需评估合规、时延与成本）。

---

## 一、视觉识别类

用于包裹分拣、车辆调度、安防监控、库容分析等。

| 场景 | 能力 | 常见 API / 服务 |
|------|------|------------------|
| 包裹分拣 | 面单条码、二维码、印刷/手写文字 | 通用 OCR、条码识别、快递面单专用 OCR（如阿里云 OCR 等） |
| 破损件 | 外包装缺陷判别 | 图像分类 / 分割 / 缺陷检测 API |
| 园区车辆 | 车牌识别、放行与登记 | 车牌识别 API、道闸 SDK |
| 装卸区安防 | 安全帽、倒地、违停等 | 目标检测服务（如 YOLO 系列云端/边缘部署） |
| 库容管理 | 货架占用、托盘空置率 | 视频分析 / 视觉计数 API，可选深度相机 |

**要点简述**

- **包裹/条码**：OCR 输出结构化字段，与 WMS/分拣控制系统对接实现自动分拣；破损件可叠加缺陷模型拦截复核。
- **车辆与人员**：出入口车牌识别；装卸区用目标检测识别违规行为并告警。
- **库容**：普通摄像头 + 视频分析 API，估算占用率与空置率，作为盘点与补货辅助。

---

## 二、自然语言处理类

主要用于**客服**、**单据录入**及**调度交互**。

| 场景 | 能力 | 常见 API / 服务 |
|------|------|------------------|
| 智能客服 | 「货到哪了」「何时派送」等高频查询；承运商与异常咨询 | 对话机器人 API（如通义**千问**等）、意图识别、实体提取；可叠加 **RAG** 知识库 |
| 语音调度 | 司机/调度员语音上报转写并录入 TMS；异常口述生成结构化日志 | **语音转文字**（如 **Whisper API**、云厂商 ASR）+ NLU 或 LLM 结构化输出 |
| 单据处理 | 非标准化运单/合同的关键字段抽取 | **NLP 信息抽取**（NER/关系抽取）、文档理解 API；复杂版式可结合 OCR |

**要点简述**

- **智能客服**：意图识别区分「查物流 / 改址 / 投诉」等；实体提取运单号、手机号、时间窗，自动调 TMS 查询接口回复，减轻人工座席。
- **语音调度**：将「货已装完，准备发车」等语句写入运输任务或状态机；异常场景转写后落库为可检索日志。
- **单据处理**：从自由文本或扫描件中抽取发货方、收货方、品名、体积/重量等，与主数据匹配后减少手工录入。

## 三、运筹优化类

### 功能路径规划 vs 复杂 VRP

- **地图路线与 ETA（功能路径规划）**  
  可调用 **高德**、**Google Directions** 等**路线规划 REST API**，获取道路级路径、**实时路况**与**分段预计时间**，用于单车导航、展示 ETA，或构建点与点之间的**时间/距离矩阵**（作为优化输入）。

- **多车、多约束（VRP / VRPTW 等）**  
  当问题涉及多车辆、容量、时间窗、装卸顺序等组合约束时，通常**不**仅靠地图 API 一次调用解决，而是建立**数学模型**，使用**优化求解器**求最优或高质量可行解，例如：
  - 商业：**Gurobi**、**IBM CPLEX**
  - 开源：**Google OR-Tools**（含 CP-SAT、路由与 VRP 示例）

本项目「智慧物流」中的**网格 A\*** 与「智慧 TMS」中的**蚁群**用于**原理演示**；工程上常见组合为：**地图 API → 成本矩阵 → 求解器（或元启发式 + 求解器回调）**。

### 其他典型场景

| 场景 | 说明 |
|------|------|
| 仓内路径 | A*、D*、时窗路径（与道路 API 无关，多为栅格/拓扑图） |
| 库存与波次 | 安全库存、波次切割、储位分配（MIP、启发式、仿真） |
| 排班排程 | 司机、月台、分拣线（约束规划、求解器） |

## 四、预测分析类

### 需求预测（时序 API · 1～7 天单量）

利用**时序预测 API**（如 **Amazon Forecast**、**Facebook Prophet** 的云服务/托管版本），基于**历史订单**聚合得到的日或小时序列，预测未来 **1～7 天**的**单量/件量**及预测区间，用于：

- **运力储备**：提前锁定车辆、司机班次、承运商额度（对接 TMS）。  
- **仓库排班**：分拣、装卸、库内人力的周计划与波次容量（对接 WMS/人力系统）。

常见形态还包括 **DeepAR**（Forecast 内置）、各云厂商的 **AutoML 时序**、或自研 Prophet 容器部署；需注意促销、节假日等**外生变量**与数据冷启动问题。

### 其他

| 场景 | 说明 |
|------|------|
| **ETA** | 时序 + 路况特征，分钟级置信带（参见「智慧 TMS」） |
| **SKU/线路** | 更细粒度货量预测，可分层或分组建模 |
| **异常预警** | 延误与破损风险、设备 PHM |
