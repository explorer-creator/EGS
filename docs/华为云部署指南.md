# EGS 华为云公网部署指南

本文档说明如何将 EGS 设备工艺管理系统部署到华为云，使其通过公网访问。

---

## 一、架构概览

```
公网用户 → 华为云 ECS (Nginx 80/443)
           ├── /        → 前端静态文件 (front/dist)
           ├── /api/*   → 代理服务 (proxy 8080)
           └── (可选) /business/* → 商业分析服务 (8001)
```

**前置条件**：
- 华为云账号
- 已购买 ECS 弹性云服务器（建议 2 核 4G 以上）
- 已绑定弹性公网 IP（EIP）

---

## 二、购买与准备 ECS

### 2.1 创建 ECS

1. 登录 [华为云控制台](https://console.huaweicloud.com/)
2. 进入 **计算** → **弹性云服务器 ECS** → **购买弹性云服务器**
3. 配置建议：
   - **镜像**：Ubuntu 22.04 或 CentOS 7.9
   - **规格**：2 vCPU、4 GB 内存（最低）
   - **系统盘**：40 GB
   - **网络**：选择有公网带宽的 VPC，或购买弹性公网 IP

### 2.2 配置安全组

确保以下端口开放：

| 端口 | 协议 | 说明        |
|------|------|-------------|
| 22   | TCP  | SSH 远程登录 |
| 80   | TCP  | HTTP        |
| 443  | TCP  | HTTPS（可选）|
| 8080 | TCP  | 代理服务（若直接访问需开放） |

路径：**安全组** → 选择 ECS 关联的安全组 → **入方向规则** → **添加规则**

### 2.3 获取公网 IP

在 ECS 列表查看 **弹性公网 IP**，记为 `你的公网IP`，后续访问地址为：`http://你的公网IP`

---

## 三、环境安装

使用 SSH 连接到 ECS 后执行（以 Ubuntu 为例）：

### 3.1 安装 Java 17

```bash
sudo apt update
sudo apt install -y openjdk-17-jdk
java -version  # 验证
```

### 3.2 安装 Node.js（用于构建前端）

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
node -v && npm -v
```

### 3.3 安装 Nginx

```bash
sudo apt install -y nginx
sudo systemctl enable nginx
```

### 3.4 安装 Python 3（可选，商业分析模块需用）

```bash
sudo apt install -y python3 python3-pip
```

## 四、部署步骤

### 4.1 上传代码

在**本地**打包并上传到 ECS（或使用 Git 拉取）：

```bash
# 方式一：Git 拉取（推荐）
ssh 你的用户名@你的公网IP
git clone https://github.com/explorer-creator/EGS.git
cd EGS
```

若使用 SCP 上传本地项目：

```bash
# 在本地执行
scp -r D:\EGS\EGSfb\* 你的用户名@你的公网IP:~/EGS/
```

### 4.2 构建前端

```bash
cd ~/EGS/front
npm install
npm run build
```

构建产物在 `front/dist/` 目录。

### 4.3 构建代理服务

```bash
cd ~/EGS/proxy
# 需预装 Maven，或从本地上传已构建的 jar
sudo apt install -y maven  # 若未安装
mvn clean package -DskipTests
```

生成的 jar 位于 `proxy/target/*.jar`（如 `xdm-f-gateway-service-1.0.0-SNAPSHOT.jar`）。

### 4.4 配置 Nginx

创建或编辑 Nginx 站点配置：

```bash
sudo nano /etc/nginx/sites-available/egs
```

写入以下内容（将 `你的公网IP` 替换为实际 IP，有域名可替换为域名）：

```nginx
server {
    listen 80;
    server_name 你的公网IP;   # 有域名可填：www.example.com
    root /home/你的用户名/EGS/front/dist;
    index index.html;

    # 前端静态资源
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理到 Spring Boot 8080
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 商业分析接口（若启用）
    location /business/ {
        proxy_pass http://127.0.0.1:8001/;
        proxy_set_header Host $host;
    }
}
```

启用配置并重载 Nginx：

```bash
sudo ln -sf /etc/nginx/sites-available/egs /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 4.5 配置生产环境变量

编辑 `proxy/src/main/resources/application.yml` 或通过环境变量覆盖（见下节）。

**重要**：部署到公网时建议开启 Mock 模式（若无 xDM-F 8003），或正确配置 `xdm.base-url` 指向你的 xDM-F 地址。

### 4.6 启动代理服务

使用 systemd 或 nohup 后台运行：

**方式一：nohup**

```bash
cd ~/EGS/proxy
nohup java -jar target/xdm-f-gateway-service-1.0.0-SNAPSHOT.jar \
  --xdm.mock-mode=true \
  --cors.allowed-origins=http://你的公网IP,http://你的公网IP:80 \
  > proxy.log 2>&1 &
```

**方式二：systemd（推荐）**

```bash
sudo nano /etc/systemd/system/egs-proxy.service
```

写入：

```ini
[Unit]
Description=EGS Proxy Service
After=network.target

[Service]
Type=simple
User=你的用户名
WorkingDirectory=/home/你的用户名/EGS/proxy
ExecStart=/usr/bin/java -jar target/xdm-f-gateway-service-1.0.0-SNAPSHOT.jar --xdm.mock-mode=true --cors.allowed-origins=*
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动并开机自启：

```bash
sudo systemctl daemon-reload
sudo systemctl enable egs-proxy
sudo systemctl start egs-proxy
sudo systemctl status egs-proxy
```

### 4.7 启动商业分析服务（可选）

若需商业分析、绿色生产等功能：

```bash
cd ~/EGS/business-analysis
pip3 install -r requirements.txt
nohup python3 run.py > business.log 2>&1 &
```

## 五、环境变量说明（生产部署）

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `XDM_MOCK_MODE` | 无 8003 时改为 `true` | `true` |
| `XDM_BASE_URL` | xDM-F 地址（有 8003 时填写） | `http://内网IP:8003/rdm_xxx_app/services` |
| `CORS_ORIGINS` | 允许的前端地址 | `http://你的公网IP,https://你的域名` |

## 六、访问与验证

1. 浏览器访问：`http://你的公网IP`
2. 登录：`admin` / `admin123`（Mock 模式）
3. 检查设备、物料、工序等页面是否正常

## 七、常见问题

### Q1：502 Bad Gateway

- 检查代理服务是否启动：`curl http://127.0.0.1:8080/api/...`
- 查看日志：`journalctl -u egs-proxy -f` 或 `tail -f proxy.log`

### Q2：前端白屏或接口 404

- 确认 `front/dist` 已正确构建
- 确认 Nginx `root` 路径正确
- 确认 `/api/` 代理到 8080

### Q3：跨域错误

- 在代理启动参数中增加：`--cors.allowed-origins=http://你的公网IP,*`
- 或修改 `application.yml` 中 `cors.allowed-origins`

### Q4：绑定域名与 HTTPS

1. 在华为云 **云解析服务** 添加 A 记录，将域名指向 ECS 公网 IP
2. 使用 Let's Encrypt 免费证书：
   ```bash
   sudo apt install certbot python3-certbot-nginx
   sudo certbot --nginx -d 你的域名
   ```
