# 仓库调度模块说明

## 功能概览

- **出仓登记**：记录货物编码、起始仓库、目的片区、重量、出仓时间（内存演示，重启代理后清空）。
- **智能调度**：在预设的「中心仓 → 中转站 → 片区」网络上，用 **Dijkstra** 求 **最小成本（公里当量）** 路径，并估算在途小时；结合历史出仓在路径节点上的货量累加，输出各节点 **压力%**、**可行性得分**、说明。
- **可视化**：网络关系图（节点大小随压力变化）、柱状图对比各节点压力与可行性。

## 演示网络（后端写死）

- 中心仓：`W1`～`W3`；中转：`H1`～`H4`；目的片区：`R1`～`R5`。
- 边带 `costKm`、`edgeCapacity`（用于演示，当前优化主目标为 `costKm`）。

## API（代理服务，前缀 `/api/warehouse-dispatch`）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/network` | 节点与边，供前端画图 |
| POST | `/outbound` | Body: `warehouseId`, `goodsCode`, `destRegionId`, `weightKg`, `outboundTime` |
| GET | `/records` | 出仓记录列表 |
| GET | `/optimize` | Query: `fromWarehouse`, `toRegion`, `goodsWeightKg` — 路径与节点分析 |
| GET | `/stress-summary` | 各非片区节点压力汇总 |

## 前端入口

顶部菜单 **「仓库调度」**，或全局搜索「仓库调度 / 出仓 / 中转 / 压力」等关键词。
