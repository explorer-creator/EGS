# 本地开发指南

本文档说明如何在本地启动前端、代理及可选后端，以及**无 8003 环境**下的 Mock 模式开发。

---

## 一、方案一：代理 Mock 模式（推荐，无需 8003）

当队友没有 xDM-F（8003）环境时，可使用代理 Mock 模式，由代理返回模拟数据，实现本地开发与联调。

### 1.1 开启 Mock 模式

**方式一：配置文件**

在 `proxy/src/main/resources/application.yml` 中设置：

```yaml
xdm:
  mock-mode: true
```

**方式二：环境变量**

```bash
# Windows PowerShell
$env:XDM_MOCK_MODE="true"

# Linux / macOS
export XDM_MOCK_MODE=true
```

### 1.2 启动步骤

1. **启动代理**

```bash
cd proxy
mvn spring-boot:run
```

2. **启动前端**

```bash
cd front
npm install   # 首次需安装依赖
npm run dev
```

3. **访问应用**

- 前端：http://localhost:5173
- 登录账号：`admin` / `admin123`

### 1.3 Mock 数据说明

| 模块       | 说明                         |
|------------|------------------------------|
| 设备       | 2 条示例设备（SMT贴片机、AOI检测机） |
| 物料       | 2 条示例物料（PCB基板、主控芯片）   |
| 工序       | 2 条示例工序（贴片、检测）         |
| 工艺路线   | 1 条示例工艺路线                 |
| 用户       | admin（admin123）               |
| 反馈箱     | 空列表，支持新增                 |

- 增删改在**内存**中生效，重启代理后恢复为初始数据
- 生产方案、PDF 导出等仍走 bypass 或 503 提示，与 Mock 模式无关

---

## 二、方案二：完整环境（含 8003）

有 xDM-F 环境时，可关闭 Mock 模式，代理将转发请求到 8003。

### 2.1 配置

在 `application.yml` 中：

```yaml
xdm:
  mock-mode: false
  base-url: http://localhost:8003   # 或实际 xDM-F 地址
```

### 2.2 启动顺序

1. 启动 xDM-F（8003）
2. 启动代理：`cd proxy && mvn spring-boot:run`
3. 启动前端：`cd front && npm run dev`
4. 访问 http://localhost:5173

---

## 三、常见问题

### Q1：Mock 模式下登录失败？

确保使用 `admin` / `admin123`，且代理已开启 `xdm.mock-mode: true`。

### Q2：生产方案 405？

生产方案接口已配置 bypass，直接返回演示方案，与 8001 是否可用无关。

### Q3：PDF 导出失败？

PDF 导出依赖 8001，不可用时返回 503 及提示，属预期行为。

### Q4：如何确认当前是否为 Mock 模式？

启动代理时，若 `xdm.mock-mode` 为 `true`，设备、物料、工序等接口将返回 Mock 数据，不请求 8003。
