# 本地开发指南

本文档说明如何在本地启动前端、代理及可选后端，以及**无 8003 环境**下的 Mock 模式开发。

---

## 一、方案一：代理 Mock 模式（推荐，无需 8003）

当队友没有 xDM-F（8003）环境时，可使用代理 Mock 模式，由代理返回模拟数据，实现本地开发与联调。

### 1.1 开启 Mock 模式

**方式一：配置文件**

在 `proxy/src/main/resources/application.yml` 中设置：

```yaml
xdm:
  mock-mode: true
```

**方式二：环境变量**

```bash
# Windows PowerShell
$env:XDM_MOCK_MODE="true"

# Linux / macOS
export XDM_MOCK_MODE=true
```

### 1.2 启动步骤

1. **启动代理**

```bash
cd proxy
mvn spring-boot:run
```

2. **启动前端**

```bash
cd front
npm install   # 首次需安装依赖
npm run dev
```

3. **访问应用**

- 前端：http://localhost:5173
- 登录账号：`admin` / `admin123`

### 1.3 Mock 数据说明

| 模块       | 说明                         |
|------------|------------------------------|
| 设备       | 2 条示例设备（SMT贴片机、AOI检测机） |
| 物料       | 2 条示例物料（PCB基板、主控芯片）   |
| 工序       | 2 条示例工序（贴片、检测）         |
| 工艺路线   | 1 条示例工艺路线                 |
| 用户       | admin（admin123）               |
| 反馈箱     | 空列表，支持新增                 |

- 增删改在**内存**中生效，重启代理后恢复为初始数据
- 生产方案、PDF 导出等仍走 bypass 或 503 提示，与 Mock 模式无关

---

## 二、方案二：完整环境（含 8003）

有 xDM-F 环境时，可关闭 Mock 模式，代理将转发请求到 8003。

### 2.1 配置

在 `application.yml` 中：

```yaml
xdm:
  mock-mode: false
  base-url: http://localhost:8003   # 或实际 xDM-F 地址
```

### 2.2 启动顺序

1. 启动 xDM-F（8003）
2. 启动代理：`cd proxy && mvn spring-boot:run`
3. 启动前端：`cd front && npm run dev`
4. 访问 http://localhost:5173

---

## 三、常见问题

### Q1：Mock 模式下登录失败？

确保使用 `admin` / `admin123`，且代理已开启 `xdm.mock-mode: true`。

### Q2：生产方案 405？

生产方案接口已配置 bypass，直接返回演示方案，与 8001 是否可用无关。

### Q3：PDF 导出失败？

PDF 导出依赖 8001，不可用时返回 503 及提示，属预期行为。

### Q4：如何确认当前是否为 Mock 模式？

启动代理时，若 `xdm.mock-mode` 为 `true`，设备、物料、工序等接口将返回 Mock 数据，不请求 8003。

### Q5：已配置千问（Qwen / DashScope）API Key，仍提示「服务暂不可用」？

1. **先分清是哪个服务**  
   - 物料、工序、工艺路线等列表若失败，多为 **xDM-F（8003）** 未启动或返回 403，与千问无关；请开 Mock 或启动 8003。  
   - 多模态报单、产品知识问答等走 **DashScope**，才与 `DASHSCOPE_API_KEY` 有关。

2. **确认 Key 已被 Java 代理加载**  
   - 设置环境变量：`DASHSCOPE_API_KEY=sk-你的Key`（PowerShell：`$env:DASHSCOPE_API_KEY="sk-..."`），**重启** `mvn spring-boot:run`。  
   - 或在 `proxy/src/main/resources/application-local.yml` 中配置 `qwen.api-key`（勿提交 Git），再重启。

3. **自检接口（代理启动后）**  
   浏览器或 curl 访问：`http://localhost:8080/api/qwen/status`  
   - `apiKeyConfigured: false`：Key 未读入，检查环境变量或 `application-local.yml`。  
   - `dashscopeReachable: false`：查看返回中的 `probeHttpStatus`、`probeErrorBody`、`hint`（401 多为 Key 错；429 为额度/限流；网络问题会报连接异常）。

4. **地址与模型**  
   - 默认：`QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1`  
   - 文本模型：`QWEN_MODEL=qwen-plus`；视觉：`QWEN_VL_MODEL=qwen-vl-plus`（需在控制台已开通）。

5. **前端**  
   - Vite 将 `/api` 代理到 `localhost:8080`，请先确认 **代理已启动**；端口被占用时若前端不是 5173，需保证 `application.yml` 里 `cors.allowed-origins` 包含当前前端地址。
