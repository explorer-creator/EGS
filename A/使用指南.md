# 工业级图片自动化获取模块 - 使用指南

本项目旨在提供工业级图片自动化获取的核心逻辑，同时包装了基于 `FastAPI` 的后端服务和基于 `Vue 3` 的前端交互组件。本指南将介绍如何配置环境、启动后端服务以及如何在已有前端项目中集成此模块。

## 1. 环境与前置准备

1. **环境依赖安装**：
   打开终端进入项目目录，安装所有必要的 Python 第三方库：
   ```bash
   pip install -r requirements.txt
   ```

2. **配置阿里云 API Key（通义万相兜底生图强烈建议配置）**：
   爬图失败或质量不达标时会调用 **DashScope 通义万相**（`ImageSynthesis`）生成工业风示意图。请在运行前配置环境变量，或在 `A` 目录复制 `.env.example` 为 `.env` 并填写 `DASHSCOPE_API_KEY`（`server.py` 会自动加载）。
   ```bash
   # Windows (CMD) 临时生效
   set DASHSCOPE_API_KEY=您的阿里云APIKEY

   # Windows (PowerShell) 临时生效
   $env:DASHSCOPE_API_KEY="您的阿里云APIKEY"

   # Linux / macOS 临时生效
   export DASHSCOPE_API_KEY="您的阿里云APIKEY"
   ```
   - 模型按顺序尝试：`wanx2.1-t2i-turbo` → `wanx2.1-t2i-plus` → `wanx-v1`（需账号在百炼/ DashScope 开通对应模型）。
   - 未配置 Key 时跳过 AI 生图，仅依赖爬图或占位图。
   - **最终兜底配图**：仓库内 `assets/materials/planetary_fallback.png`（行星减速器示意图）。爬取与通义均失败时返回该图，避免纯色占位。

---

## 2. 启动 FastAPI 后端服务

我们已经封装好了基于 `FastAPI` 的高性能异步后端服务 `server.py`。该服务会在 `8000` 端口开启 RESTful API，并内置了图片静态文件挂载。

**启动服务**（主站「智能物料配图」依赖本服务，**必须先启动再点生成**）：
```bash
cd EGSfb/A
pip install -r requirements.txt
python server.py
# 或者使用 uvicorn 直接启动
uvicorn server:app --host 0.0.0.0 --port 8000 --reload
```

**自检**：浏览器打开 `http://127.0.0.1:8000/` 应返回 `{"status":"ok",...}`。若前端仍报 **500**，在浏览器 F12 → Network 查看 `generate-image` 请求：应对 **127.0.0.1:8000**（经 Vite 代理为 `/api/module-a/...`），而不是 Java 的 8080。

**API 接口规范说明**：
- **请求方式**：`POST`
- **请求地址**：`http://localhost:8000/api/generate-image`
- **请求体 (JSON)**：
  ```json
  {
      "keyword": "液压泵"
  }
  ```
- **成功响应示例 (JSON)**：
  ```json
  {
      "status": "ok",
      "keyword": "液压泵",
      "image_url": "/output/液压泵_1710000000.jpg",
      "filename": "液压泵_1710000000.jpg"
  }
  ```
  *(注：返回的 `image_url` 为相对路径，前端组件会自动拼接基础域名并进行渲染。通过此相对路径访问静态文件服务器便可获取图片。)*

## 3. 前端交互界面使用 (Vue 3)

项目中提供了一个即插即用的前端交互组件：`MaterialGenerator.vue`。这个 Vue 3 组件采用了 Composition API (组合式 API) 语法，内置了深/浅色模式自适应以及现代玻璃质感的设计效果。

**集成步骤**：
1. **应用组件**：将 `MaterialGenerator.vue` 复制到您的 Vue 项目的 `src/components/` 目录下（或相应位置），并在您的页面中引入：
   ```vue
   <template>
     <div>
       <!-- 其他页面内容 -->
       <MaterialGenerator />
     </div>
   </template>

   <script setup>
   import MaterialGenerator from './components/MaterialGenerator.vue'
   </script>
   ```

2. **配置后端地址**：
   组件代码内部默认向 `http://localhost:8000` 发起 API 请求。如果您将后端服务部署到了不同的 IP 或外网域名上，请在您 Vue 前端项目根目录下的 `.env` 文件中配置变量：
   ```env
   VITE_API_BASE=http://您的后端实际域名:8000
   ```

3. **样式调优**：
   该组件内置了诸如加载骨架屏、状态流转过渡等完整的交互样式，并且它的部分样式属性利用了 CSS 变量定义（如 `var(--color-background)`）。它会自动随您的主项目发生颜色更替；如需个性化修改色彩参数，也可直接微调组件内 `<style scoped>` 代码中的 CSS 映射。

## 4. 执行流程与注意事项

1. **执行响应时间**：
   用户在前端页面输入关键字触发生成后，后端会使用多线程在网络并发爬取候选图片并进行裁剪白边、清晰度验证与肤色过滤等处理。如果这套流水线 10 秒内未获得理想图片或遭遇连续重复拦截，由于流程自动升级至 Aliyun 图像生成系统，整个过程耗时可能会提升至 10~15 秒（前端内部网络超时设定为 30 秒，超过该时间便会提醒异常重试）。
   
2. **物料直接落盘**：
   无论是网络爬取合规图片，还是利用 AI 最终补救生成的图片，在发回给前端响应前，文件均已经被安全落地到项目同目录下的 `output/` 文件夹中持久化存储。

3. **应对高并发架构**：
   目前项目采用了线程池 `ThreadPoolExecutor` 挂载于 FastAPI 事件循环下，以应对异步环境中的阻塞调用，非常适合课设或中低强度的并发请求。若进入大规模工业级高并发部署场景，建议引入如 Redis + Celery 等队列服务接管繁重的生图流程。

## 5. 与丝路智联主前端（Vue 2 + Vite）的拼接

仓库内 **`EGSfb/front`** 已集成模块 A：

1. **启动顺序**：先在本机启动本目录 `python server.py`（端口 **8000**），再启动 Java 代理与 `npm run dev`。
2. **Vite 代理**：开发时将前缀 **`/api/module-a`** 转发到 `http://127.0.0.1:8000`，并重写路径：
   - `POST /api/module-a/api/generate-image` → FastAPI 的 `/api/generate-image`
   - `GET /api/module-a/output/xxx.jpg` → 静态目录 `/output/xxx.jpg`
3. **页面入口**：顶部菜单 **「智能物料配图」**，组件为 `front/src/components/MaterialGenerator.vue`（Vue 2 Options API + axios）。
4. **仓库内仍保留** `A/MaterialGenerator.vue`（Vue 3 版）可作参考；以 `front/src` 下版本为准。

生产环境部署时，请将 FastAPI 与前端 Nginx 做同级反向代理，勿把 Python 服务暴露公网无鉴权。
